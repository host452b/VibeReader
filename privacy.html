VibeReader Privacy Policy

Last updated: 2026-03-21

1. What Data We Collect

VibeReader processes the following data locally on your device:

  • Page content — visible text, title, URL, and metadata of web pages you choose to analyze
  • User preferences — language, selected AI provider, model, prompt templates (stored in Chrome sync storage)
  • API key — your AI provider credentials (stored only in local device storage, never synced to cloud)

We do not collect analytics, telemetry, browsing history, or any data beyond what you explicitly interact with.

2. How Data Is Transmitted

When you use a cloud AI provider, page content and your prompt are sent to the provider's API endpoint via HTTPS. This only happens when you explicitly click "Send" or enable Auto Summary.

Local providers (Ollama, LM Studio) process everything on your machine. No data leaves your device.

4. Your Control

  • Choose local-only: Select Ollama or LM Studio — zero data leaves your device
  • Review before sending: Use the RAW_TXT editor to inspect extracted content before any API call
  • Disable Auto Summary: Turn off in Settings to prevent automatic API calls
  • Delete all data: Uninstall the extension or clear extension data in Chrome settings

5. Third Parties

VibeReader does not use analytics, advertising, or tracking services. No data is shared with any party other than your chosen AI provider.

6. Contact

For questions about this policy, please open an issue on the project repository.

VibeReader プライバシーポリシー

最終更新日: 2026-03-21

1. 収集するデータ

VibeReaderは以下のデータをお使いのデバイス上でローカルに処理します：

  • ページコンテンツ — 分析対象ページの表示テキスト、タイトル、URL、メタデータ
  • ユーザー設定 — 言語、AIプロバイダ、モデル、プロンプトテンプレート（Chrome同期ストレージに保存）
  • APIキー — AIプロバイダの認証情報（ローカルデバイスストレージのみに保存、クラウドには同期されません）

アナリティクス、テレメトリ、閲覧履歴、その他ユーザーが明示的に操作する以外のデータは一切収集しません。

2. データの送信

クラウドAIプロバイダを使用する場合、ページコンテンツとプロンプトがHTTPS経由でプロバイダのAPIエンドポイントに送信されます。これは「送信」ボタンをクリックするか、自動要約を有効にした場合にのみ発生します。

ローカルプロバイダ（Ollama、LM Studio）はすべてお使いのマシン上で処理します。データはデバイスの外に出ません。

対応クラウドプロバイダ

英語版の表をご参照ください。各プロバイダのプライバシーポリシーへのリンクが記載されています。

3. データストレージ

データ保存先クラウド同期
APIキーchrome.storage.localなし — デバイスのみ
設定（言語、プロバイダ、モデル）chrome.storage.syncあり — Chrome同期経由
プロンプトテンプレートchrome.storage.syncあり — Chrome同期経由
ページコンテンツメモリのみなし — 閉じると破棄

4. ユーザーの権利

  • ローカルのみ: OllamaまたはLM Studioを選択 — データはデバイス外に出ません
  • 送信前の確認: RAW_TXTエディタで抽出内容をAPI送信前に確認できます
  • 自動要約の無効化: 設定でオフにすると自動API呼び出しを停止します
  • 全データの削除: 拡張機能をアンインストールするか、Chromeの設定で拡張機能データをクリア

5. 第三者

VibeReaderはアナリティクス、広告、トラッキングサービスを使用しません。選択したAIプロバイダ以外の第三者とデータを共有することはありません。

6. お問い合わせ

このポリシーに関するご質問は、プロジェクトリポジトリでissueを作成してください。

VibeReader 隐私政策

最后更新: 2026-03-21

1. 我们收集的数据

VibeReader 在您的本地设备上处理以下数据：

  • 页面内容 — 您选择分析的网页的可见文本、标题、URL 和元数据
  • 用户偏好 — 语言、AI 提供商、模型、提示模板（存储在 Chrome 同步存储中）
  • API 密钥 — 您的 AI 提供商凭据（仅存储在本地设备，绝不同步到云端）

我们不收集分析数据、遥测数据、浏览历史或任何您未明确交互的数据。

2. 数据如何传输

当您使用云端 AI 提供商时，页面内容和提示会通过 HTTPS 发送到提供商的 API 端点。这仅在您点击"发送"或启用自动摘要时发生。

本地提供商（Ollama、LM Studio）在您的机器上处理所有内容，数据不会离开您的设备。

支持的云端提供商

请参阅英文版的完整列表，其中包含各提供商隐私政策的链接。

3. 数据存储

数据存储位置是否同步到云端
API 密钥chrome.storage.local否 — 仅限本机
偏好设置（语言、提供商、模型）chrome.storage.sync是 — 通过 Chrome 同步
提示模板chrome.storage.sync是 — 通过 Chrome 同步
页面内容仅内存否 — 关闭即销毁

4. 您的控制权

  • 选择纯本地模式: 选择 Ollama 或 LM Studio — 数据完全不离开设备
  • 发送前审查: 使用 RAW_TXT 编辑器在 API 调用前检查提取的内容
  • 禁用自动摘要: 在设置中关闭以防止自动 API 调用
  • 删除所有数据: 卸载扩展或在 Chrome 设置中清除扩展数据

5. 第三方

VibeReader 不使用任何分析、广告或追踪服务。除您选择的 AI 提供商外，不与任何第三方共享数据。

6. 联系方式

如对本政策有疑问，请在项目仓库中创建 issue。
